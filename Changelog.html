📝 NSBEdit — Standalone RTF Notepad

 

Latest Update: 16 May 2026 — v2026.05.16.09 (09:58)
Lightweight portable RTF editor for Windows. Drop the exe anywhere and run.

NSBEdit Version 2026.05.16.09

RichEdit line-number gutter, autocomplete fix (16.05.2026, 09:58):

  • RichEdit line-number gutter (NsbLineGutter): A custom child window class is now registered and painted alongside each RichEdit tab. It is always visible — as a thin strip (toggle button only) when line numbers are off, and at full width when on. Clicking the strip toggles numbers on all open tabs simultaneously.
  • Ne_EnsureLineGutter(): Creates the NsbLineGutter window for a RichEdit doc if it does not already exist, and attaches a tooltip that shows “Show line numbers” or “Hide line numbers” depending on current state.
  • Ne_SyncRichGutters(): Repositions all gutters and shrinks the editor rect to make room after NeTabs_SetRects. Scintilla tabs receive only the thin strip (Scintilla draws its own built-in margin); RichEdit tabs receive full width (S(44)) when on, thin strip (S(20)) when off.
  • Ne_SyncLineNumBtn() rewritten: Iterates all open tabs — sends SCI_SETMARGINWIDTHN to Scintilla windows and invalidates / updates the tooltip text on RichEdit gutters. Calls Ne_SyncRichGutters at the end to keep layout consistent.
  • s_lineNumsOn global flag: Persists across tab switches. Ne_SetupScintillaStyle reads it to set the initial Scintilla margin width when a new Scintilla window is created.
  • NeTabDoc::hLineGutter field: Added to NeTabDoc to store the companion gutter HWND per tab.
  • Autocomplete case-insensitive: SCI_AUTOCSETIGNORECASE TRUE added to Ne_SetupScintillaStyle.

NSBEdit Version 2026.05.15.16

Syntax highlighting, AltGr fix, FTP Save to FTP (15.05.2026, 16:56):

  • Syntax highlighting working for all languages: New Ne_ApplyLang(hSci, langIdx) wrapper sends the keyword list via SCI_SETKEYWORDS so the lexer can colour keywords, and applies per-lexer style overrides where Scintilla’s generic style indices do not apply. Previously no keywords were ever sent to the lexer, so all text remained black.
  • PHP switched to phpscript lexer: PHP was mapped to the hypertext (HTML-embedded) lexer whose PHP token style IDs start at 118 — outside the generic 1–10 range. Switched to phpscript with correct SCE_HPHP_* style IDs (118–127): strings red, keywords blue/bold, $variables purple, comments green italic, numbers green.
  • Language menu converts RichEdit tab to Scintilla: Choosing a language on an untitled (RichEdit) tab now extracts the plain text, creates a Scintilla window at the same position, loads the text in, hides the RichEdit, and applies the chosen lexer and styles — no reopen needed.
  • AltGr key fix (Norwegian / international keyboards): Windows reports AltGr as Ctrl+Alt. The main message loop’s Ctrl-shortcut block was intercepting AltGr+0 (‘}’) and similar before they reached the editor. Fixed by skipping the shortcut block when Right Alt (VK_RMENU) is held.
  • File → Save to FTP…: Uploads the active document to any connected FTP server. The FTP browser opens in save mode — a filename edit box above the tree, and “Save here” (green) / “Cancel” (red) buttons at the bottom. After upload the tab is marked FTP-linked, a green success dialog confirms, and the connection stays open. Temp file written to %TEMP%\NSBEdit_savelocal\ for the upload.
  • Locale additions: MENU_SAVE_TO_FTP, FTP_SAVE_BROWSER, FTP_FILENAME_PROMPT, FTP_SAVE_HERE, FTP_SAVE_TO, FTP_SAVED_OK added to locale/en_GB.txt.

NSBEdit Version 2026.05.14.10

Hyperlink insertion, Ctrl+Click to follow links, hover tooltip, API/INTERNALS rebrand (14.05.2026, 10:51):

  • Insert Hyperlink dialog: Owner-draw dialog with URL and display-text fields, Tab/Shift+Tab navigation, Save (blue) and Cancel (red) owner-draw buttons. Inserts an RTF \field{\fldinst{HYPERLINK “url”}}{\fldrslt display} field at the caret. Pre-fills URL or display text if the current selection looks like a URL.
  • URL validation with regex: The Save button validates the URL against a std::wregex pattern requiring a recognised scheme (https:// http:// ftp(s):// mailto: file:// or bare www.), at least one dot-separated host label, and a 2–4 alpha-character TLD. Malformed URLs show a custom NSBEdit warning dialog (warning icon + OK button) rather than a system MessageBox.
  • Ctrl+Click to follow hyperlinks: ENM_LINK added to all EM_SETEVENTMASK calls. Ne_EditCaretProc subclass intercepts WM_LBUTTONDOWN when VK_CONTROL is held and the cursor is IDC_HAND; the URL is extracted from the RTF field instruction (Ne_ExtractLinkUrlAt streams out RTF around the click, parses HYPERLINK “…”), then opened with ShellExecuteW.
  • Hover tooltip on hyperlinks: Ne_EditCaretProc WM_MOUSEMOVE detects IDC_HAND cursor (set by RichEdit over links). On cursor entering a link: extracts the URL and shows a two-line ShowMultilingualTooltip — line 1 is the URL, line 2 is [Ctrl]+Click to open in browser. Tooltip hides when the cursor leaves the link area or the editor.
  • Custom warning dialog for validation errors: Ne_ShowChoiceDialog extended with optional hMsgIcon parameter. Warning and OK-button icons loaded from system (IDI_WARNING) and shell32.dll (index 294). Dialog font upgraded from DEFAULT_GUI_FONT to 12pt Segoe UI (Ne_CreateDialogFont) and stored in NeDialogData::hDlgFont, deleted on WM_NCDESTROY.
  • API/INTERNALS rebrand: All SetupCraft references in every API_INTERNALS/API/*.txt and API_INTERNALS/INTERNALS/*.txt file replaced with NSBEdit.
  • Link dialog shell fixes: static bool registered guard (replaces fragile GetClassInfoW check), COLOR_WINDOW+1 background, WM_CTLCOLORSTATIC handler, AdjustWindowRectEx sizing, button height S(34).
  • Locale additions: LINK_TIP_CTRL, MSG_LINK_BAD_URL, BTN_OK added to locale/en_GB.txt.

Table properties dialog — Apply/Cancel working, alter vs nested mode (11.05.2026, 13:11):

  • Apply button now inserts/alters table correctly: Control values (rows, columns, column width, borders, padding, row height, alignment) are read inside the window procedure before DestroyWindow destroys the child HWNDs, then stored in a module-level NeTableProps struct. The calling function uses that struct directly — no more dead HWNDs returning zero.
  • Alter vs Nested mode: When the caret is inside an existing table, a radio group appears at the top of the dialog: Alter current table (pre-selected) or Table in current cell. Alter mode scans adjacent \intbl paragraphs around the caret to select the whole table, then streams in the replacement RTF. Nested mode inserts a fresh table at the caret without touching the surrounding table.
  • Buttons centred: Apply (blue) and Cancel (red) owner-draw buttons are now laid out centred horizontally in the dialog rather than right-aligned.
  • Fully localised: TBLP_MODE_ALTER and TBLP_MODE_NESTED added to locale/en_GB.txt; all new UI text goes through Ls().

NSBEdit Version 2026.05.11.09

Owner-draw menus, tabbed editor, 12pt UI fonts, status bar with icons (11.05.2026, 09:59):

  • 12pt UI fonts everywhere: All user-visible text — toolbar buttons, combos, dialogs, status bar, tooltips — now uses 12pt Segoe UI rendered DPI-aware via GetDpiForWindow()/GetDpiForSystem(). Previously the default system message font was used, which Windows 10/11 regressed to a too-small size.
  • Owner-draw popup menus with white background: All popup menu items (File/Edit/Help dropdowns) are now owner-drawn at 12pt Segoe UI. White background on normal items, system highlight colour on hover, correct grayed colour for disabled items. Separators drawn as thin light-grey lines. Accelerator text right-aligned.
  • Owner-draw menu bar items: File, Edit, and Help bar entries are also owner-drawn at 12pt via Ne_AppendMenuOD(isBar=true), fixing the too-small system default.

A [+] button opens a new tab. Right-click context menu on the tab strip: New Tab / Close Tab (localized). Tab titles use non-breaking spaces as padding so the × glyph is drawn by WM_PAINT.
  • Custom status bar — ne_statusbar.h/cpp: Owner-drawn status bar showing Words: N and Chars: N on the left, and a shell32 icon + Saved (green checkmark, index 294) or Unsaved (red X, index 131) on the right. Updates in real-time on every EN_CHANGE. Bold 12pt font.
  • Ctrl+W behaviour: With more than one tab, closes the active tab (with save prompt). With exactly one tab remaining, closes the application.
  • Softer dialog button colours with hover: Owner-draw dialog buttons use a muted green/red/blue palette. A subclass proc tracks mouse enter/leave so the hover tint updates without polling.
  • Locale additions: TAB_CTX_NEW_TAB, TAB_CTX_CLOSE_TAB, SB_WORDS, SB_CHARS, SB_SAVED, SB_UNSAVED added to locale/en_GB.txt.

NSBEdit Version 2026.05.11.08

Custom decision dialogs with icon buttons and external-file change detection (11.05.2026, 08:38):

  • Unsaved-changes prompt is now a custom NSBEdit dialog: Replaced the system MessageBoxW(MB_YESNOCANCEL) flow with a dedicated modal dialog using owner-draw buttons and localized labels. Buttons are Save, Don't Save, and Cancel, each with a small icon for clearer visual feedback.
  • Focus-based external change detection: When the editor regains focus, NSBEdit compares file stamp (last write time + size) against the stamp captured when focus was last lost. If the file changed on disk, a custom decision dialog asks whether to Reload from disk or Keep Current.
  • Reusable custom dialog infrastructure: Added a shared Ne_ShowChoiceDialog path with dynamic text measurement, dynamic button-width sizing, centered layout, and owner-window modal behavior.
  • Safer load pipeline: Added Ne_LoadPathIntoEditor as a single code path used by Open, startup file argument, and external reload flow. This keeps editor state and title/status updates consistent.
  • Locale updates: Added new keys for dialog titles, prompts, and button labels so all new UI text remains fully localized through Ls().

NSBEdit Version 2026.05.10.15

First release — standalone portable RTF notepad (10.05.2026, 15:15):

  • Core editor: Full RTF formatting toolbar — Bold, Italic, Underline, Strikethrough, Subscript, Superscript, Font face, Font size, Text colour, Highlight colour, Align Left/Centre/Right/Justify, Bullet list, Numbered list, Insert image. RichEdit50W (Msftedit.dll) with fallback to RichEdit20W. BS_AUTOCHECKBOX | BS_PUSHLIKE toggle buttons sync state with EN_SELCHANGE. Image insertion via \pict\pngblip / \jpegblip hex stream.
  • File menu: New, Open, Save, Save As, Print, Exit — with unsaved-change prompt (Save before closing?) on New/Open/Exit. RTF streamed in/out via EM_STREAMIN / EM_STREAMOUT. File filter includes RTF and plain text.
  • Status bar: Shows current line, column, and full file path (or “Untitled”). Updated on every EN_CHANGE / EN_SELCHANGE.
  • About dialog: GDI+ logo rendering from NSB.png, version and publish date read from curver.txt (written by NewVersion.ps1). Subtitle, description, and licence notice all localised.
  • Licence dialog: Full GPLv2 text streamed from GPLv2.md into a read-only RichEdit. Button to open the About dialog.
  • Single portable exe: Statically linked (-static -static-libgcc -static-libstdc++), no external DLLs beyond Windows system ones. Drop anywhere and run — no installer, no registry, no .ini file.
  • DPI-aware: PerMonitorV2 manifest, all pixel values scaled through S() via g_dpiScale. App icon loaded at correct small and large sizes via LoadImageW + WM_SETICON.
  • i18n — locale/en_GB.txt embedded as RCDATA (resource ID 10): No external locale files at runtime. Ne_LoadLocale() loads the resource at startup; Ls(key) returns the localised string (falls back to the key itself). Ne_Filter(key) builds double-null file-dialog filter strings from |-delimited locale values. Ne_Unescape() expands \n → CRLF, \t → tab, \\ → backslash in locale values. Every MessageBoxW, OPENFILENAMEW filter / title, menu string, tooltip, and dialog string goes through Ls().
  • Standalone dependencies — dpi.h / dpi.cpp / tooltip/tooltip.h / tooltip/tooltip.cpp: Copied locally into the project folder — no parent-directory references. Shared tool modules live in subfolders (tooltip/, scroll/). The project builds entirely from its own folder with a single .\makeit.bat.
  • One-by-one toolbar reflow — Ne_LayoutToolbar: Greedy left-to-right placement: 17 controls are laid out in sequence; when the next control would overflow the current row it drops to the next row — one control at a time. This gives a smooth, predictable reflow as the window is narrowed, not a sudden jump between one-row and two-row states.
  • Minimum window width — WM_GETMINMAXINFO: Ne_LayoutToolbar computes the minimum client width analytically (the widest row at the current layout, stored in NeState::minClientW). WM_GETMINMAXINFO converts this to a minimum window width via AdjustWindowRectEx and enforces it via ptMinTrackSize.x. The window cannot be narrowed below the point where any row would hold fewer than three controls.
  • Hover tooltips on all 17 toolbar controls: Subclass proc (Ne_TipSubclassProc) attached via Ne_SetTip. Tip text stored as a heap-allocated window property (neTipText), freed on WM_NCDESTROY. Uses the project’s ShowMultilingualTooltip / HideTooltip system.
  • Build system: makeit.bat — kills the running instance, compiles the resource with windres, links everything (NSBEdit.cpp dpi.cpp tooltip.cpp NSBEdit.res) with g++ -std=c++17 -O2 -mwindows -municode, outputs NSBEdit.exe. NewVersion.ps1 writes curver.txt with the current timestamp (Published: DD.MM.YYYY HH:mm, Version: YYYY.MM.DD.HH).
