Lightweight portable RTF editor for Windows. Drop the exe anywhere and run.

NSBEdit Version 2026.05.18.09

Paragraph Spacing dialog restyled and fixed (18.05.2026, 09:08):

  • Paragraph Spacing dialog restyled and fixed: The Paragraph Spacing dialog (Format → Paragraph Spacing) now uses the app-standard owner-draw button system (NeBtnTone / NeDialogButtonSpec / Ne_DrawDialogButton / Ne_BtnHoverProc) with a white background, a blue Save button and a red Cancel button, and hover highlight on both. Root cause of Save/Cancel doing nothing fixed: the dialog class previously used DefWindowProcW as its WndProc, which silently discarded the WM_COMMAND notification sent synchronously by button clicks — the GetMessageW loop never saw it. The dialog now has a dedicated Ne_ParSpaceDlgProc that reads the spin-box values on IDOK, stores them in module-level statics (s_parSpBefPt / s_parSpAftPt), and calls DestroyWindow; the message loop exits when IsWindow(dlg) returns false, and the values are applied immediately after. All strings go through Ls() (i18n-correct: DLG_PARSPACE, DLG_PARSPACE_BEF, DLG_PARSPACE_AFT, BTN_SAVE, BTN_CANCEL).

NSBEdit Version 2026.05.17.15

Horizontal Rule, status bar “Rich text” (17.05.2026, 15:07):

  • Horizontal Rule (HR) in RTF documents: HR paragraphs render as a custom-drawn line across the editor in one of six styles — single, thick, double, dotted, dashed, or hairline. Colour (solid or gradient), width %, and left/right indent are configurable via a properties dialog. Core functions: Ne_InsertHRule, Ne_PaintHRules, Ne_RebuildHRList (g_hrMap HWND→entry cache), Ne_DeleteHRule, Ne_ShowHRulePropsDialog. Drawing is overlaid via GetDC after CallWindowProc in WM_PAINT to avoid nested-BeginPaint clip conflicts. HR behaves like a character that occupies its whole line: pressing Delete at the end of the line above or Backspace at the start of the line below removes it; pressing Enter on any line above an HR moves it one line down; Ctrl+Z correctly restores a deleted HR. Three bugs fixed during development: (a) typing on a line above the HR used to draw the HR through the typed text — EN_CHANGE → Ne_RebuildHRList now keeps charIdx current at all times; (b) pressing Enter above the HR used to erase it — NE_WM_HR_CLEANUP (posted via PostMessageW) now receives enterPos in wParam and strips only the paragraph at that position if it inherited HR format, keeping the undo chain intact and avoiding re-entrant EM_EXSETSEL calls during mid-split RichEdit processing; (c) Ctrl+Z after deleting an HR now correctly redraws it — the EN_CHANGE guard that skipped Ne_RebuildHRList when g_hrMap was empty has been removed.
  • Status bar shows “Rich text” on RTF/RichEdit tabs alongside the word and character count.

NSBEdit Version 2026.05.16.14

HTML5 export, RTF toolbar on startup, open-in-same-tab (16.05.2026, 14:32):

  • Export as HTML 5… (Convert menu, RTF documents only): Converts the active RTF document to a self-contained HTML5 file with all images embedded as base64 data URIs. Uses a bundled rtf2html library (rtf2html/ne_rtf2html_lib.cpp) with several fixes: \*\picprop groups inside \pict are now correctly skipped so their property-name text (containing hex-like letters) cannot corrupt the image data; char_by_code() in rtf_tools.h now converts Windows-1252 code points to proper UTF-8 (2- or 3-byte sequences) instead of emitting raw bytes, so all Norwegian and other non-ASCII characters render correctly. The output is <!DOCTYPE html> UTF-8 with a <meta viewport>, inline CSS from the RTF style table, and img { max-width:100%; height:auto; }. Menu item is greyed when the active tab is not RTF.
  • App opens with RTF toolbar on startup: Ne_DocIsRtf() now returns true for untitled tabs that have a RichEdit window and no Scintilla window, so the Rich Text formatting toolbar is visible and all RTF-only Convert menu items are enabled as soon as the app launches — before any file is opened or saved.
  • File → Open reuses untitled tab: When File → Open is used and the active tab is an untouched untitled RichEdit tab (no path, not modified, no Scintilla), the file loads directly into that tab instead of creating an extra blank tab first.

Even a single connected profile now requires an explicit click — useful for deliberately saving a file to a different server than it came from. NSBEdit Version 2026.05.16.10

Custom autocomplete popup, phrase & keyword completion, makeit.bat fix (16.05.2026, 10:47):

  • Custom autocomplete popup (ne_autocomplete/): A new NsbAutoComplete window class (CS_DROPSHADOW, WS_EX_NOACTIVATE | WS_EX_TOPMOST) replaces Scintilla’s built-in SCI_AUTOCSHOW for all completion modes. The popup matches the NSBEdit tooltip style — system tooltip yellow background, dark amber border, muted sage green selection highlight with white text. Font is DPI-aware 12pt Segoe UI via GetDpiForWindow and MulDiv.
  • Phrase completion working: Scintilla’s built-in autocomplete cancelled itself whenever the entered text contained non-word characters (spaces, =, $, etc.). The custom popup has no such restriction. When the trimmed line prefix contains a space, the whole document is scanned for lines that begin with the same prefix (case-insensitive), deduplicated, sorted, and shown — up to 30 candidates.
  • Keyword completion: Keywords and language built-ins now also appear in the same custom popup, giving a consistent look for both completion modes.
  • Up to 9 visible items with ▲/▼ scroll arrows and mouse wheel scrolling when the list is longer.
  • Keyboard: ↑/↓ navigate; Tab or Enter accept; Escape dismiss; Backspace/Delete dismiss and pass through to Scintilla. Tab/Enter acceptance uses a pendingAccept flag — the WM_KEYDOWN is consumed and the next WM_CHAR is swallowed before it reaches Scintilla, so no stray tab character or newline is ever inserted.
  • Mouse click acceptance: WM_MOUSEACTIVATE → MA_NOACTIVATE in the popup proc, plus a WM_KILLFOCUS guard in the Scintilla subclass (skip dismiss if focus went to the popup itself), ensures that clicking a list item always inserts the completion correctly.
  • Insertion guard: g_acInserting flag prevents autocomplete from re-triggering on each SCN_CHARADDED notification fired during SCI_REPLACESEL insertion.
  • Position and layout: Popup appears below the caret line and flips above if it would extend past the bottom of the monitor work area.
  • makeit.bat fix: taskkill output is now shown so build failures are visible. A 1-second timeout /NOBREAK added after the kill prevents the linker from failing with “Permission denied” when the running NSBEdit.exe has not yet released its file handle.

NSBEdit Version 2026.05.15.16

Syntax highlighting, AltGr fix, FTP Save to FTP (15.05.2026, 16:56):

  • Syntax highlighting working for all languages: New Ne_ApplyLang(hSci, langIdx) wrapper sends the keyword list via SCI_SETKEYWORDS so the lexer can colour keywords, and applies per-lexer style overrides where Scintilla’s generic style indices do not apply. Previously no keywords were ever sent to the lexer, so all text remained black.
  • PHP switched to phpscript lexer: PHP was mapped to the hypertext (HTML-embedded) lexer whose PHP token style IDs start at 118 — outside the generic 1–10 range. Switched to phpscript with correct SCE_HPHP_* style IDs (118–127): strings red, keywords blue/bold, $variables purple, comments green italic, numbers green.
  • Language menu converts RichEdit tab to Scintilla: Choosing a language on an untitled (RichEdit) tab now extracts the plain text, creates a Scintilla window at the same position, loads the text in, hides the RichEdit, and applies the chosen lexer and styles — no reopen needed.
  • AltGr key fix (Norwegian / international keyboards): Windows reports AltGr as Ctrl+Alt. The main message loop’s Ctrl-shortcut block was intercepting AltGr+0 (‘}’) and similar before they reached the editor. Fixed by skipping the shortcut block when Right Alt (VK_RMENU) is held.
  • File → Save to FTP…: Uploads the active document to any connected FTP server. The FTP browser opens in save mode — a filename edit box above the tree, and “Save here” (green) / “Cancel” (red) buttons at the bottom. After upload the tab is marked FTP-linked, a green success dialog confirms, and the connection stays open. Temp file written to %TEMP%\NSBEdit_savelocal\ for the upload.
  • Locale additions: MENU_SAVE_TO_FTP, FTP_SAVE_BROWSER, FTP_FILENAME_PROMPT, FTP_SAVE_HERE, FTP_SAVE_TO, FTP_SAVED_OK added to locale/en_GB.txt.

NSBEdit Version 2026.05.10.15

First release — standalone portable RTF notepad (10.05.2026, 15:15):

  • Core editor: Full RTF formatting toolbar — Bold, Italic, Underline, Strikethrough, Subscript, Superscript, Font face, Font size, Text colour, Highlight colour, Align Left/Centre/Right/Justify, Bullet list, Numbered list, Insert image. RichEdit50W (Msftedit.dll) with fallback to RichEdit20W. BS_AUTOCHECKBOX | BS_PUSHLIKE toggle buttons sync state with EN_SELCHANGE. Image insertion via \pict\pngblip / \jpegblip hex stream.
  • File menu: New, Open, Save, Save As, Print, Exit — with unsaved-change prompt (Save before closing?) on New/Open/Exit. RTF streamed in/out via EM_STREAMIN / EM_STREAMOUT. File filter includes RTF and plain text.
  • Status bar: Shows current line, column, and full file path (or “Untitled”). Updated on every EN_CHANGE / EN_SELCHANGE.
  • About dialog: GDI+ logo rendering from NSB.png, version and publish date read from curver.txt (written by NewVersion.ps1). Subtitle, description, and licence notice all localised.
  • Licence dialog: Full GPLv2 text streamed from GPLv2.md into a read-only RichEdit. Button to open the About dialog.
  • Single portable exe: Statically linked (-static -static-libgcc -static-libstdc++), no external DLLs beyond Windows system ones. Drop anywhere and run — no installer, no registry, no .ini file.
  • DPI-aware: PerMonitorV2 manifest, all pixel values scaled through S() via g_dpiScale. App icon loaded at correct small and large sizes via LoadImageW + WM_SETICON.
  • i18n — locale/en_GB.txt embedded as RCDATA (resource ID 10): No external locale files at runtime. Ne_LoadLocale() loads the resource at startup; Ls(key) returns the localised string (falls back to the key itself). Ne_Filter(key) builds double-null file-dialog filter strings from |-delimited locale values. Ne_Unescape() expands \n → CRLF, \t → tab, \\ → backslash in locale values. Every MessageBoxW, OPENFILENAMEW filter / title, menu string, tooltip, and dialog string goes through Ls().
  • Standalone dependencies — dpi.h / dpi.cpp / tooltip/tooltip.h / tooltip/tooltip.cpp: Copied locally into the project folder — no parent-directory references. Shared tool modules live in subfolders (tooltip/, scroll/). The project builds entirely from its own folder with a single .\makeit.bat.
  • One-by-one toolbar reflow — Ne_LayoutToolbar: Greedy left-to-right placement: 17 controls are laid out in sequence; when the next control would overflow the current row it drops to the next row — one control at a time. This gives a smooth, predictable reflow as the window is narrowed, not a sudden jump between one-row and two-row states.
  • Minimum window width — WM_GETMINMAXINFO: Ne_LayoutToolbar computes the minimum client width analytically (the widest row at the current layout, stored in NeState::minClientW). WM_GETMINMAXINFO converts this to a minimum window width via AdjustWindowRectEx and enforces it via ptMinTrackSize.x. The window cannot be narrowed below the point where any row would hold fewer than three controls.
  • Hover tooltips on all 17 toolbar controls: Subclass proc (Ne_TipSubclassProc) attached via Ne_SetTip. Tip text stored as a heap-allocated window property (neTipText), freed on WM_NCDESTROY. Uses the project’s ShowMultilingualTooltip / HideTooltip system.
  • Build system: makeit.bat — kills the running instance, compiles the resource with windres, links everything (NSBEdit.cpp dpi.cpp tooltip.cpp NSBEdit.res) with g++ -std=c++17 -O2 -mwindows -municode, outputs NSBEdit.exe. NewVersion.ps1 writes curver.txt with the current timestamp (Published: DD.MM.YYYY HH:mm, Version: YYYY.MM.DD.HH).
